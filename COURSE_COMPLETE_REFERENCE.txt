KEY CODE PATTERN:
  # Tool definition (JSON schema)
  record_user_details_json = {
      "name": "record_user_details",
      "description": "Use this tool to record that a user is interested...",
      "parameters": {
          "type": "object",
          "properties": {
              "email": {"type": "string", "description": "..."},
              "name":  {"type": "string", "description": "..."},
          },
          "required": ["email", "name"]
      }
  }
  tools = [{"type": "function", "function": record_user_details_json}]

  # Elegant dispatch (no if-else chain):
  def handle_tool_calls(tool_calls):
      for tool_call in tool_calls:
          tool_name = tool_call.function.name
          arguments = json.loads(tool_call.function.arguments)
          tool = globals().get(tool_name)
          result = tool(**arguments) if tool else {}

  # The agentic loop:
  while not done:
      response = openai.chat.completions.create(
          model="gpt-4o-mini", messages=messages, tools=tools)
      if response.choices[0].finish_reason == "tool_calls":
          messages += handle_tool_calls(response.choices[0].message.tool_calls)
      else:
          done = True

  src/<project_name>/
    crew.py              — @CrewBase class with @agent, @task, @crew methods

AGENTS.YAML EXAMPLE:
  researcher:
    role: Senior Financial Researcher for {company}
    goal: Research the company, news and potential for {company}
    backstory: You're a seasoned financial researcher...
    llm: openai/gpt-4o-mini

  analyst:
    role: Market Analyst and Report writer focused on {company}
    goal: Analyze company {company} and create a comprehensive report
    backstory: You're a meticulous, skilled analyst...
    llm: openai/gpt-4o-mini

KEY CODE PATTERN (crew.py):
  from crewai import Agent, Crew, Process, Task
  from crewai.project import CrewBase, agent, crew, task
  from crewai_tools import SerperDevTool

  @CrewBase
  class ResearchCrew():
      agents_config = 'config/agents.yaml'
      tasks_config  = 'config/tasks.yaml'

      @agent
      def researcher(self) -> Agent:
          return Agent(config=self.agents_config['researcher'],
                       tools=[SerperDevTool()])

      @agent
      def analyst(self) -> Agent:
          return Agent(config=self.agents_config['analyst'])

      @task
      def research_task(self) -> Task:
          return Task(config=self.tasks_config['research_task'])

      @task
      def analysis_task(self) -> Task:
          return Task(config=self.tasks_config['analysis_task'],
                      output_file='output/report.md')

      @crew
      def crew(self) -> Crew:
          return Crew(agents=self.agents,
                      tasks=self.tasks,
                      process=Process.sequential)

RUN COMMAND:
  $ crewai run

OUTPUT: output/report.md

OVERVIEW:

  conditional routing, and is pure Python — no LLM required to understand
  the fundamentals.

  "### A word about 'Annotated'
   LangGraph needs us to use this feature when we define our State object.
   It wants us to tell it what function it should call to update the State
   with a new value. This function is called a reducer.
   LangGraph provides a default reducer called add_messages which takes
   care of the most common case."

KEY CODE PATTERN:
  from langgraph.graph import StateGraph, START, END
  from langgraph.graph.message import add_messages
  from langchain_openai import ChatOpenAI
  from pydantic import BaseModel
  from typing import Annotated

  class State(BaseModel):
      messages: Annotated[list, add_messages]

  graph_builder = StateGraph(State)
  llm = ChatOpenAI(model="gpt-4o-mini")

  def chatbot_node(old_state: State) -> State:
      response = llm.invoke(old_state.messages)
      return State(messages=[response])

  graph_builder.add_node("chatbot", chatbot_node)
  graph_builder.add_edge(START, "chatbot")
  graph_builder.add_edge("chatbot", END)
  graph = graph_builder.compile()

  # Visualize:
  display(Image(graph.get_graph().draw_mermaid_png()))

  from autogen_core import AgentId, MessageContext, RoutedAgent, message_handler
  from autogen_core import SingleThreadedAgentRuntime
  from dataclasses import dataclass

  @dataclass
  class Message:
      content: str

  class SimpleAgent(RoutedAgent):
      def __init__(self):
          super().__init__("Simple Agent")

      @message_handler
      async def on_my_message(self, message: Message, ctx: MessageContext) -> Message:
          return Message(content=f"I disagree with: {message.content}")

  runtime = SingleThreadedAgentRuntime()
  await SimpleAgent.register(runtime, "simple_agent", lambda: SimpleAgent())
  runtime.start()
  response = await runtime.send_message(
      Message("hello"), AgentId("simple_agent", "default"))

  PART 9: CAN GOOGLE GEMINI FREE API REPLACE EVERYTHING IN THIS CODEBASE?
================================================================================

SHORT ANSWER:
  Partially — Gemini can replace the raw LLM calls and most LLM-dependent
  features, but several parts of the codebase depend on OpenAI-specific
  infrastructure (SDK features, hosted tools, tracing) that have no direct
  Gemini free equivalents.

  The codebase actually already uses Gemini in multiple places (Labs 2, 3),
  so the integration path is well-established!

------------------------------------------------------------------------
GOOGLE GEMINI FREE TIER — WHAT YOU GET
------------------------------------------------------------------------
  Models available free (via Google AI Studio):
    - gemini-1.5-flash          (fast, high rate limits)
    - gemini-2.0-flash          (best free model, 15 RPM, 1500 RPD)
    - gemini-2.5-flash          (used already in this course)
    - gemini-2.5-pro             (limited free quota)

  Free tier limits (Google AI Studio):
    - 15 requests per minute (RPM)
    - 1,000,000 tokens per minute (TPM)
    - 1,500 requests per day (RPD)
    — These limits can make looped/concurrent agent systems slow or blocked

  API Access:
    Native Gemini SDK:
      from google.generativeai import GenerativeModel

    OpenAI-compatible endpoint (already used in this codebase in Lab 2, 3):
      gemini = OpenAI(
          api_key=google_api_key,
          base_url="https://generativelanguage.googleapis.com/v1beta/openai/"
      )

------------------------------------------------------------------------
MODULE-BY-MODULE GEMINI COMPATIBILITY ANALYSIS
------------------------------------------------------------------------

MODULE 1 — FOUNDATIONS:
  STATUS: FULLY COMPATIBLE ✓
  - Lab 2 already uses Gemini via the OpenAI-compatible endpoint.
  - Lab 3 already uses Gemini as the evaluator (gemini-2.5-flash).
  - All raw API call patterns work with Gemini.
  - REPLACEMENT: Replace OpenAI() with OpenAI(api_key=..., base_url=gemini_url)
    or use google-generativeai SDK directly.

MODULE 2 — OPENAI AGENTS SDK:
  STATUS: PARTIALLY COMPATIBLE ⚠
  - Lab 3 already shows how to plug Gemini into the Agents SDK using
    OpenAIChatCompletionsModel wrapper — this works.
  - Lab 4 uses WebSearchTool — this is an OpenAI HOSTED tool. Gemini has
    Google Search grounding but it's not integrated into the Agents SDK.
    You can replace WebSearchTool with SerperDevTool or Brave MCP server
    as your search mechanism.
  - Tracing (platform.openai.com/traces) is OpenAI-specific and won't
    work with Gemini models, but the agents themselves will run.
  - REPLACEMENT: Use OpenAIChatCompletionsModel(openai_client=gemini_async_client)
    and replace WebSearchTool with a custom search tool.

MODULE 3 — CREWAI:
  STATUS: COMPATIBLE WITH CHANGES ⚠
  - CrewAI uses litellm under the hood, which supports Gemini via:
      llm: google/gemini-2.0-flash
    in agents.yaml
  - Install: pip install crewai[google]
  - SerperDevTool still works (separate from the LLM)
  - LIMITATION: Some CrewAI default behaviors assume OpenAI structure;
    check CrewAI docs for Gemini-specific configuration.
  - REPLACEMENT: Change llm: openai/gpt-4o-mini
                 to    llm: google/gemini-2.0-flash in agents.yaml

MODULE 4 — LANGGRAPH:
  STATUS: FULLY COMPATIBLE ✓
  - LangGraph is LLM-agnostic. The LLM is just an invoke() call.
  - Install: langchain-google-genai (already in requirements)
  - Replace: ChatOpenAI(model="gpt-4o-mini")
    with:    ChatGoogleGenerativeAI(model="gemini-2.0-flash")
  - All graph structure, checkpointing, and tool patterns remain identical.
  - LangSmith tracing is provider-agnostic.
  - with_structured_output() works with Gemini models.
  - REPLACEMENT: pip install langchain-google-genai
    from langchain_google_genai import ChatGoogleGenerativeAI
    llm = ChatGoogleGenerativeAI(model="gemini-2.0-flash")

MODULE 5 — AUTOGEN:
  STATUS: COMPATIBLE WITH CHANGES ⚠
  - AutoGen supports Gemini via google-generativeai extension or via the
    OpenAI-compatible endpoint:
      from autogen_ext.models.openai import OpenAIChatCompletionClient
      # Using Gemini's OpenAI-compatible endpoint:
      model_client = OpenAIChatCompletionClient(
          model="gemini-2.0-flash",
          base_url="https://generativelanguage.googleapis.com/v1beta/openai/",
          api_key=google_api_key,
      )
  - MCP tools in AutoGen (Lab 2) are provider-agnostic.
  - gRPC distributed runtime (Lab 4) is LLM-agnostic.
  - REPLACEMENT: Point OpenAIChatCompletionClient to Gemini endpoint.

MODULE 6 — MCP:
  STATUS: FULLY COMPATIBLE ✓ (with rate limit caution)
  - MCP is a protocol standard — completely LLM-agnostic.
  - The OpenAI Agents SDK wrapping is the only LLM-dependent part.
  - Already shown in Lab 3: OpenAIChatCompletionsModel allows Gemini.
  - All MCP servers (fetch, Playwright, filesystem, memory, Brave, Polygon)
    work identically regardless of which LLM sits on top.
  - The trading floor (Lab 5) can run with Gemini — it has a USE_MANY_MODELS
    flag specifically designed to let you mix providers.
  - RATE LIMIT WARNING: The trading floor runs 4+ agents in parallel loops.
    Free tier (15 RPM) will cause rate limit errors. Stagger calls or use
    the paid Gemini API for production use.
  - REPLACEMENT: Set USE_MANY_MODELS=True and configure gemini models,
    or replace gpt-4o-mini defaults in traders.py.

------------------------------------------------------------------------
THINGS GEMINI FREE CANNOT REPLACE
------------------------------------------------------------------------

  1. OPENAI WEBSEARCHTOOL (Module 2, Lab 4):
     The WebSearchTool is an OpenAI-hosted tool billed per use ($0.025/call).
     Gemini has Google Search grounding, but it's not in OpenAI Agents SDK.
     WORKAROUND: Use SerperDevTool (needs SERPER_API_KEY, has free tier),
     Brave Search MCP (needs BRAVE_API_KEY, has free tier), or
     DuckDuckGo search via LangChain community tools (no API key needed).

  2. OPENAI TRACES UI (platform.openai.com/traces):
     The trace() context manager and trace viewer are OpenAI-specific.
     WORKAROUND: Use LangSmith (langsmith.com) for tracing — it works
     with any LLM and is supported across Modules 4 and 5.

  3. RATE LIMITS IN PRODUCTION:
     Free tier (15 RPM) is insufficient for the trading floor's parallel
     4-agent concurrent execution. You will hit 429 errors.
     WORKAROUND: Use Gemini paid API, or use free-tier Groq (very fast,
     generous limits) as a fallback model for some agents.

  4. OPENAI VECTOR STORES (FileSearchTool, Module 2):
     The FileSearchTool uses OpenAI's proprietary vector store service.
     No direct Gemini equivalent.
     WORKAROUND: Use LangChain with Chroma or FAISS as a local vector
     store + embeddings (can use Google's embedding-001 model for free).

  5. OPENAI COMPUTER USE (ComputerTool):
     Anthropic and OpenAI both have computer use. Gemini does not yet
     offer an equivalent tool in this SDK context.

FINAL VERDICT
------------------------------------------------------------------------

  Google Gemini free tier is an EXCELLENT replacement for ~80-85% of
  this codebase with minimal code changes. The integration pattern
  (OpenAI-compatible endpoint) is already demonstrated in the course.

  The remaining 15-20% that requires workarounds:
    - WebSearchTool → use DuckDuckGo or Brave Search (both free)
    - OpenAI Traces → use LangSmith (free tier available)
    - Rate limits in the trading floor → throttle calls or use paid tier
    - FileSearchTool/ComputerTool → use local vector stores instead

  The course is excellently designed for provider independence — every
  framework covered (LangGraph, AutoGen, CrewAI, MCP) abstracts away the
  underlying LLM. This makes Gemini substitution relatively painless.
